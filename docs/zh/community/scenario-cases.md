# 场景案例

场景案例帮助用户理解 DataFoundry 如何处理真实感数据 Agent 任务，包括 schema 检查、只读 SQL、证据追溯和最终经营决策。

## 什么是好的案例

好的案例应足够具体、可复现，也适合对外展示：

- 可识别的业务或运营场景。
- 小规模、可公开演示的数据集或 schema 轮廓。
- 清晰的分析路径，包括用户会提出的问题。
- 可以从数据中验证的预期发现。
- 关于治理、隐私、权限控制或生产边界的说明。

[DTC 增长经营复盘](../examples/dtc-growth-demo.md) 是当前参考案例。它把订单、广告投放、商品、客服工单和目标数据组合成一个可追溯的经营复盘。

## 如何提交建议

通过 [GitHub issue](https://github.com/datagallery-lab/datafoundry/issues) 提交：

- 案例标题和目标读者。
- 业务问题或工作流目标。
- 涉及的数据表、文件、API 或外部系统。
- 预期分析步骤和代表性输出。
- 需要匿名化、mock 或排除的内容。

案例不应依赖私有数据访问。如果真实场景比较敏感，请描述工作流形态，并提供脱敏数据集或合成数据等价物。
