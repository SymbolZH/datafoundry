# 协作原则

DataFoundry 是面向可治理数据 Agent 工作流的开源项目。社区协作应该让项目更容易理解、复现和运营。

## 原则

- **证据优先：** 缺陷反馈、功能建议和场景提案都应包含足够上下文，方便复现或评估。
- **区分已落地与计划中：** 文档和讨论应清楚区分已经发布的能力和路线图想法。
- **不共享敏感数据：** 不要分享密钥、凭据、私有数据集、客户记录或专有业务数据。
- **重视可追溯性：** 尽量提供 SQL、日志、trace 输出、截图或命令，说明结果是如何产生的。
- **小改动更容易评审：** 一个 pull request 聚焦一个行为、一个文档改进或一个场景。

## 沟通方式

请使用直接、技术化的描述。好的反馈会说明发生了什么、预期应该是什么，以及如何复现差异。

对于新想法，请描述用户角色、工作流、数据源、预期输出，以及当前产品界面为什么还不能覆盖它。
